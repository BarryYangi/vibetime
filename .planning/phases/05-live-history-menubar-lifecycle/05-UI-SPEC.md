# Phase 05 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-phase, verified inline for Phase 5 planning.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | coss registry via `components.json` (`@coss`) |
| Preset | coss neutral, light/dark semantic tokens |
| Component library | Base UI via coss local components |
| Icon library | lucide-react |
| Font | SN Pro for sans/heading; system mono stack for timers and numeric data |
| Animation library | Motion package, imported from `motion/react`; do not use legacy `framer-motion` imports |
| Charts | ECharts v6 using registered `cossNeutral` theme |

### Component Rules

- Reuse existing local coss components before adding new primitives.
- Live and History routes must be added to the existing HashRouter/sidebar shell.
- History Top Projects must use the local coss `Table`.
- Menubar right-click is native Electron menu, not a renderer-styled menu.
- Renderer app surfaces must keep the current neutral theme and must not reintroduce Tokyo Night styling.
- Interactive app controls use the default arrow cursor rather than the hand pointer cursor. Keep text-entry cursors for inputs, but do not use `cursor-pointer` for buttons, switches, badges, links, tabs, or menu-like controls.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, table micro-gaps, active indicator offsets |
| sm | 8px | Compact button gaps, badge spacing, small chart labels |
| md | 16px | Default control spacing, card/table padding |
| lg | 24px | View section padding, chart block gaps |
| xl | 32px | Live stage gaps, dashboard row gaps |
| 2xl | 48px | Major Live display separation |
| 3xl | 64px | Reserved for full-screen Live single-turn composition only |

Exceptions: Electron traffic-light offsets and native menubar positioning may use platform-specific pixel values; chart internals may use ECharts numeric spacing when required for label fit.

### Layout Contracts

- App shell keeps `min-width: 960px` and `min-height: 640px` unless planning explicitly changes the Electron window contract.
- Live view must fit one active turn without scrolling at 960x640. Multiple active turns may use vertical compression; all active turns remain simultaneously visible.
- History view may scroll vertically, but each chart block needs stable min-height so loading, empty, and populated states do not shift surrounding sections.
- Menubar dropdown must be compact enough for a native menu-bar popover. It is not a full Today replacement.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.45 |
| Label | 11px | 500 | 1.25 |
| Heading | 24px | 600 | 1.15 |
| Dashboard Section Title | 15px | 600 | 1.25 |
| Live Project Name | 40-72px responsive by container constraints, not viewport units | 700 | 0.95-1.05 |
| Live Timer | 56-112px mono, tabular, constrained to fit container | 700 | 0.9-1.0 |
| Chart Axis / Table Meta | 11-12px | 400-500 | 1.2 |
| Menubar Dropdown Text | 12-13px | 400-600 | 1.25 |

### Typography Rules

- Do not scale font size directly with viewport width.
- Use tabular mono for elapsed timers, durations, totals, and menubar-equivalent numeric displays.
- Live display text must use container constraints and wrapping/truncation so long project names do not overlap timers or active-turn metadata.
- History headings stay compact. Do not use Live-scale type inside History cards or tables.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / neutral 950 in dark | Main app surfaces |
| Secondary (30%) | `var(--card)`, `var(--muted)`, `var(--border)` | Chart panels, tables, side surfaces |
| Accent (10%) | `var(--primary)`, `var(--info)`, `var(--success)`, `var(--warning)` | Active pulse, chart series, selected range, current-turn indicators |
| Destructive | `var(--destructive)` | Quit/destructive affordances only when a confirmation or danger state exists |

Accent reserved for: active Live indicator, Motion sweep, chart series, selected period/highlight, active menubar dot, status badges. Do not color every button or table row with accent hues.

### Chart Palette

- Use `cossNeutral` theme as the base.
- History stacked area uses Top 5 projects plus Others. Keep total series count <= 6.
- Calendar heatmap uses a neutral-to-primary intensity ramp; avoid rainbow heatmaps.
- Tooltips should use semantic surface/background values and readable 12px-13px text.

## Screen Contracts

### Live View

- First viewport is the active stage, not a dashboard grid.
- One active turn: full-stage composition with project name, agent/model, timer, pulse, sweep, today's project total footer.
- Multiple active turns: vertical sections; all sections visible simultaneously at the minimum window size.
- Idle: ambient subtle motion plus idle copy. It must not show the Today project list.
- Motion should animate transform/opacity where possible. Avoid expensive layout animation loops.
- Respect reduced-motion if a platform/browser signal is available; fall back to static pulse/sweep states.

### History View

- Single-page analytics dashboard.
- Period selector options: `7d`, `30d`, `90d`, `365d`.
- 365-day ECharts native calendar heatmap is always present.
- Selected period filters applicable charts/tables and can highlight the corresponding date range on the heatmap.
- Stacked area chart shows Top 5 projects plus Others.
- Top Projects table columns: Project, Total, Turns, Last Active. Table must be sortable.

### Menubar Widget

- Title states: idle `●`, idle with total `<dot> 47m`, and `● 5h 23m` when >= 1h.
- Active turn state is indicated by the dot/title state; native pulsing is not required for V0 because Electron native Tray no longer exposes a supported highlight/pulse API.
- Left click and right click both open the same native Electron menu.
- Native menu contains Today, running-turn status rows, Top project rows, Open, Settings, and Quit.
- Section labels are disabled/grey with no icon; concrete turn/project rows are clickable and use native macOS template icons.
- Menubar title refreshes on DB push and at visible minute-boundaries while active turns exist; avoid high-frequency polling in the resident app.

### Lifecycle & Auto-launch

- Window close hides the main window and keeps menubar alive.
- Quit only via menubar Quit or Cmd+Q.
- Open restores last active view; first-run default is Today.
- Settings includes `Open at login` toggle.
- First opt-in prompt appears after the user has context, such as first successful Settings open or first successful agent connection.

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| coss local components | Button, Card, Badge, Table, Input, Separator, Spinner; add Popover/Dialog/Switch only if needed via `@coss` registry | Verify imports against local files or coss docs before use |
| shadcn official | none planned | not required |
| third-party blocks | none | not allowed without explicit planning note |

### Dependency Safety

- Add `motion` if Live uses Motion components. Imports must be from `motion/react`.
- Do not add another charting library.
- Do not add another UI component library.
- Native menubar behavior should use Electron APIs instead of renderer overlays.
