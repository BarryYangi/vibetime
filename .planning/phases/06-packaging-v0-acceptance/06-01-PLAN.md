# Phase 06 Plan 01: Packaging, Signing & V0 Acceptance

**Status:** In Progress
**Phase:** 06-packaging-v0-acceptance
**Requirements:** DIST-01, DIST-02, plus V0 acceptance checklist

## Goal

Produce a macOS arm64 `.app` and `.dmg`, ad-hoc sign the app, bundle `vibetime-hook` correctly, document first launch, and verify V0 acceptance from the packaged artifact.

## Plan Summary

This phase is intentionally one integrated plan. Packaging, hook path resolution, signing, and acceptance are coupled; splitting them risks producing a `.dmg` before the app can actually install hooks from inside the bundle.

## Verification Gate

Phase 6 is complete only if all are true:

- Root CI passes.
- `.app` exists.
- `.dmg` exists.
- bundled `vibetime-hook` exists and is executable.
- ad-hoc signing is verified.
- packaged app launches.
- settings install path writes packaged hook path.
- V0 acceptance checklist has no unresolved blockers.

## Known Constraints

- Notarization is out of scope.
- Auto-update is out of scope.
- Linux/Windows builds are out of scope.
- Development-mode Open at Login verification is not final; packaged app verification is required.

## Rollback

- Packaging config can be removed without touching runtime data.
- Hook install tests must use temp HOME or restore backups to avoid altering user configs.
- No migration changes are planned.
