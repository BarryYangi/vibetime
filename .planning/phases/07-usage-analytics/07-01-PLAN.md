<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local transcript fixtures -> core parser contracts | Fixture JSONL can contain sensitive prompt/response/tool strings; contracts must prevent persistence. |
| Core schema -> desktop SQLite init | DDL constants define what can ever be stored locally. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-usage-privacy | Information Disclosure | `usage_records` DDL and `types.ts` | mitigate | Allowlist persisted fields, forbid content-bearing column names, add canary tests for `SECRET_PROMPT_CANARY`, `SECRET_RESPONSE_CANARY`, `SECRET_TOOL_ARG_CANARY`. |
| T-scan-integrity | Tampering | `usage_records` uniqueness | mitigate | Require `UNIQUE(agent, source_file_key, source_row_key)` and scan-state keys `(agent, source_file_key)`. |
| T-pricing-integrity | Tampering | `usage_pricing_cache` | mitigate | Store validated public pricing fields only; no opaque raw JSON blob used as trusted cost truth. |
| T-attribution-integrity | Tampering | `UsageAttributionMethod` | mitigate | Enumerate attribution methods and keep unmatched rows representable. |
| T-hook-hot-path | Denial of Service | hook package | mitigate | This plan modifies `core` contracts only and does not touch `packages/hook/src/hook.ts` or `packages/hook/src/store.ts`. |
</threat_model>
