<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local filesystem -> desktop main service | Scanner reads local transcript files read-only and may encounter sensitive/malformed rows. |
| Public pricing network -> desktop main service | LiteLLM pricing payload is public but untrusted and can fail or drift. |
| Desktop service -> SQLite | Upserts must be idempotent and privacy-preserving. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-usage-privacy | Information Disclosure | `usage-service.ts` persistence | mitigate | Persist scanner facts only; tests dump usage tables and search canaries/forbidden keys. |
| T-ipc-validation | Tampering | future IPC consumers | mitigate | Service exposes typed args but IPC validation is implemented in Plan 05. |
| T-pricing-integrity | Tampering | pricing refresh | mitigate | Single LiteLLM source, shape validation, cache fallback, nullable unknown cost. |
| T-scan-integrity | Tampering | scan state/upsert | mitigate | `mtimeMs`/`size` state plus unique row keys; repeated scans keep row count stable. |
| T-attribution-integrity | Tampering | reconciliation before summary | mitigate | Preserve unmatched rows and attribution method/confidence in persisted facts. |
| T-hook-hot-path | Denial of Service | hook package | mitigate | Hook regression tests plus static scan for scanner/pricing imports in `packages/hook/src`. |
</threat_model>
