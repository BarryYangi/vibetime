<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer -> main IPC | Renderer sends untrusted Usage query/filter/refresh args. |
| Config TOML -> background timer | User-editable config can contain invalid cadence values. |
| App lifecycle -> background scanner | Background scanning must not block startup or hook invocation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-ipc-validation | Tampering | `ipc-handlers.ts` | mitigate | Strict period/agent/filter/boolean validation before service calls. |
| T-pricing-integrity | Tampering | `refreshUsage` IPC | mitigate | Refresh errors return explicit pricing state and do not erase cached values. |
| T-hook-hot-path | Denial of Service | `main/index.ts`, hook files | mitigate | Scanner starts from desktop lifecycle only; static hook scan verifies no scanner/pricing imports. |
| T-usage-privacy | Information Disclosure | IPC result types | mitigate | IPC types expose aggregated facts and sanitized rows only; no transcript content fields. |
| T-scan-integrity | Tampering | config refresh interval | mitigate | Enum validation with default `30m`; unsupported values ignored. |
</threat_model>
